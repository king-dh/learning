# 学生教育管理系统 — 前端转 Java 后端学习计划

## 适用人群
- 有 Vue3 / JavaScript 开发经验的前端工程师
- 想通过项目实战学习 Java Spring Boot 后端
- 目标是达到能独立开发前后端完整项目的初级 Java 水平

## 学习总览

```
阶段一：Java 基础（3~5 天）      ← 必须先学
阶段二：数据库基础（2~3 天）     ← 必须先学
阶段三：Spring Boot 入门（3~5 天）← 必须先学
阶段四：结合项目深入学习（持续）  ← 边看项目边学
```

## 阶段一：Java 语言基础（3~5 天）

> 目标：能看懂 Java 代码，理解基本语法，不需要手写复杂逻辑。

### 1.1 Java 和 JavaScript 的核心区别

| 概念 | JavaScript | Java |
|------|-----------|------|
| 运行环境 | 浏览器/Node.js | JVM（Java 虚拟机） |
| 类型系统 | 动态类型 `let x = 1` | 静态类型 `int x = 1` |
| 编译 | 解释执行 | 先编译成 `.class` 再执行 |
| 类 | ES6 class（语法糖） | 一切都是 class |
| import | `import { x } from './file'` | `import com.path.Class;` |
| 数组 | `[1, 2, 3]` | `new int[]{1, 2, 3}` 或 `List.of(1,2,3)` |

### 1.2 必学知识点

| 知识点 | 对应项目的代码 | 推荐学习时间 |
|--------|---------------|------------|
| **基本类型** (`int`, `long`, `String`, `boolean`) | 所有 Entity/DTO 的字段声明 | 30 分钟 |
| **类和对象** (`class`, `new`, `this`) | 所有 `*.java` 文件 | 1 小时 |
| **方法** (`public`, `void`, `return`) | Controller/Service 中的方法 | 30 分钟 |
| **接口** (`interface`, `implements`) | `StudentService.java`（接口）+ `StudentServiceImpl.java`（实现） | 1 小时 |
| **泛型** (`List<T>`, `Result<T>`) | `Result.java`, `IPage<StudentVO>` | 30 分钟 |
| **注解** (`@Override`, `@Service`) | 每个文件顶部的 `@` 标记 | 1 小时 |
| **包和 import** | 每个文件第 1 行和 3~18 行 | 20 分钟 |
| **访问修饰符** (`public`, `private`) | `private final StudentService` | 20 分钟 |

### 1.3 学习方法

1. 打开项目中的 `Student.java`（Entity 层），对照注释逐行理解
2. 打开 `StudentController.java`，理解"类 → 字段 → 方法"的结构
3. 重点理解：**Java 里一切都是类**，没有独立函数，所有代码都在 `class {}` 里面

### 1.4 推荐资源
- 菜鸟教程 Java 基础：https://www.runoob.com/java/java-tutorial.html
- 重点看：基本语法 → 对象和类 → 修饰符 → 接口 → 泛型

## 阶段三：Spring Boot 入门（3~5 天）

> 目标：理解 Spring Boot 的核心理念 — IoC 容器、依赖注入、分层架构。

### 3.1 核心概念（必须先理解）

#### IoC 容器（控制反转）
```
传统方式（你自己 new）：           Spring 方式（容器管理）：
  StudentService s = new            @Service
    StudentServiceImpl()             class StudentServiceImpl { ... }
                                     ↑ Spring 帮你 new，你只管用
```

#### 依赖注入（DI）
```
不用 DI：                          用 DI（构造器注入）：
  class Controller {                @RequiredArgsConstructor
    Service s = new ServiceImpl()   class Controller {
  }                                   private final Service s
                                    }
                                    ↑ Spring 自动把 Service 传进来
```

#### 分层架构
```
┌─ Controller 层 ── 接收 HTTP 请求，返回 JSON       （接线员）
├─ Service 层    ── 写业务逻辑，调 Mapper           （干活的人）
├─ Mapper 层     ── 操作数据库（SQL）               （数据库操作）
├─ Entity 层     ── 数据库表 → Java 对象映射        （数据表模型）
├─ DTO 层        ── 前端传过来的 JSON → Java 对象    （请求数据）
└─ VO 层         ── Java 对象 → 返回前端的 JSON     （响应数据）
```

### 3.2 必学知识点

| 知识点 | 对应项目的代码 | 推荐时间 |
|--------|---------------|---------|
| **@SpringBootApplication** | `BackendApplication.java` | 20 分钟 |
| **@RestController** / **@RequestMapping** | 所有 Controller 文件 | 30 分钟 |
| **@Service** / **@RequiredArgsConstructor** | 所有 Service 文件 | 30 分钟 |
| **@Data** (Lombok) | Entity/DTO/VO 的第一行注解 | 20 分钟 |
| **application.yml** 配置 | `application.yml` | 30 分钟 |
| **Spring Security + JWT** | `SecurityConfig.java`, `JwtUtil.java` | 2 小时 |
| **全局异常处理** | `GlobalExceptionHandler.java` | 30 分钟 |
| **事务 @Transactional** | Service 实现类的方法上 | 20 分钟 |

### 3.3 学习方法

1. 先看 `pom.xml`，了解依赖结构（Spring Boot / MyBatis-Plus / Security / JWT）
2. 看 `BackendApplication.java`，理解启动流程
3. 挑一个功能（如"分页查询学生"），从 Controller → Service → Mapper → SQL 走一遍
4. 用 Postman 或 curl 发请求，观察 Controller 如何接收参数、Service 如何查询、Mapper 如何生成 SQL

### 3.4 推荐资源
- Spring Boot 官方快速开始：https://spring.io/quickstart
- MyBatis-Plus 官方文档（看条件构造器部分）：https://baomidou.com/pages/10c804/

## 阶段四：结合本项目深入学习

> 这时候你已经有了基础，可以按以下顺序深入阅读项目代码。

### 4.1 推荐阅读顺序

### 4.2 验证学习效果的方式

1. **改需求**：比如把学生查询的每页 10 条改成默认 20 条

3. **加接口**：新增一个"按性别统计学生数量"的接口
4. **Debug**：在 Service 方法里打断点或用 `System.out.println` 调试

## 遇到问题怎么办

1. **先看注释**：项目中每个 Java 文件都已经加了详细的中文注释
2. **看日志**：后端启动后在终端可以看到每次请求的 SQL 和参数
3. **用 Postman**：浏览器访问 `http://localhost:8088/swagger-ui.html` 可以调试所有接口
4. **JS 类比**：遇到不懂的概念，用 JS 类比来理解（项目中已标注）
